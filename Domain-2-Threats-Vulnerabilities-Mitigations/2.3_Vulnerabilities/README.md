# 2.3 Vulnerability Types

A vulnerability is a weakness in a system or its design that can be exploited by a threat.

## Software Vulnerabilities
* **Zero-Day:** A vulnerability that is unknown to the vendor and has no available patch.
* **Buffer Overflow:** An attack that sends more data to a memory buffer than it can hold, potentially executing malicious code.
* **SQL Injection:** Inserting malicious SQL statements into entry fields to manipulate a database.

## Infrastructure Vulnerabilities
* **Legacy Systems:** Old hardware or software that is no longer supported or patched.
* **Firmware:** Often overlooked code embedded in hardware (routers, printers, IoT) that can contain vulnerabilities.
* **Misconfigurations:** Default passwords, open ports, or unnecessary services running on a server.

## Vulnerability Research
* **CVE (Common Vulnerabilities and Exposures):** A list of publicly disclosed cybersecurity vulnerabilities.
* **CVSS (Common Vulnerability Scoring System):** A numerical score (0.0 to 10.0) representing the severity of a vulnerability.
